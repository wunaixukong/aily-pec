# AI 智能分化训练助手 - 前端开发指南

## 一、项目背景

这是一个帮助健身者管理训练分化的 App。核心痛点：**断练不重置**。

传统健身 App 按日期安排训练（周一胸、周二背），如果周二没练，周三打开会显示"周三应该练的内容"，导致进度混乱。

本应用采用**队列式进度指针**：计划为 [胸, 背, 腿, 肩]，周一练完胸，周二周三加班，周四打开依然指向"背"。

### 3.2 分化计划接口

#### 创建训练计划
```http
POST /api/plans
Content-Type: application/json

{
  "userId": 1,
  "name": "PPL三分化",
  "workoutDays": [
    {"dayOrder": 1, "content": "推 - 胸+肩+三头"},
    {"dayOrder": 2, "content": "拉 - 背+二头"},
    {"dayOrder": 3, "content": "腿 - 股四+腘绳+小腿"}
  ]
}
```

**响应示例：**
```json
{
  "id": 3,
  "userId": 1,
  "name": "PPL三分化",
  "isActive": false,
  "workoutDays": [
    {"id": 7, "dayOrder": 1, "content": "推 - 胸+肩+三头", "createTime": "..."},
    {"id": 8, "dayOrder": 2, "content": "拉 - 背+二头", "createTime": "..."},
    {"id": 9, "dayOrder": 3, "content": "腿 - 股四+腘绳+小腿", "createTime": "..."}
  ],
  "createTime": "...",
  "updateTime": "..."
}
```

#### 获取用户的所有计划
```http
GET /api/plans/user/{userId}
```

#### 获取当前激活的计划
```http
GET /api/plans/active/{userId}
```

#### 激活指定计划
```http
PUT /api/plans/{planId}/activate?userId={userId}
```

**注意：** 用户同时只能有一个激活的计划，激活新计划会自动取消旧计划的激活状态。

#### 删除计划
```http
DELETE /api/plans/{planId}
```

---

### 3.3 今日训练接口（核心）

#### 获取今天该练什么
```http
GET /api/today/{userId}
```

**响应示例：**
```
推 - 胸+肩+三头
```

**错误情况：**
- 没有激活的计划 → 返回 500 错误 "没有激活的训练计划"
- 进度指针未初始化 → 返回 500 错误 "进度指针未初始化"

#### 初始化进度指针
```http
POST /api/today/{userId}/init
Content-Type: application/json

{
  "planId": 3
}
```

**调用时机：** 首次激活计划后，需要初始化进度指针。

#### 完成今日训练（推进指针）
```http
POST /api/today/{userId}/complete
```

**响应示例：**
```
拉 - 背+二头
```

**说明：** 完成当前训练后，指针自动推进到下一项，并返回下次训练内容。如果到达最后一项，会循环回到第一项。

## 四、前端页面建议

### 4.1 首页（今日训练）- 核心页面

**功能：**
- 显示"今天该练什么"（大字突出显示）
- "完成"按钮，点击后推进指针
- 显示上次训练日期
- 如果没有激活计划，提示创建或激活计划

**交互流程：**
```
打开App → 调用 GET /api/today/{userId}
         ↓
    显示训练内容
         ↓
    用户点击"完成"
         ↓
    调用 POST /api/today/{userId}/complete
         ↓
    显示"已完成"，并预览下次训练内容
```

- 删除计划

**创建计划表单建议：**
```

第1天: [推 - 胸+肩+三头]

第3天: [腿 - 股四+腘绳+小腿]
[+ 添加训练日]

[保存计划]
```

### 4.3 用户设置页

**功能：**
- 用户注册/登录

- 退出登录

## 五、核心业务流程

### 首次使用流程
```
1. 注册/登录
   ↓
2. 创建第一个训练计划
   ↓
3. 激活该计划
   ↓
4. 初始化进度指针（自动或手动）
   ↓
5. 进入首页，看到"今天该练什么"
```

### 日常使用流程
```
1. 打开App → 显示今天该练什么
   ↓
2. 去健身房训练
   ↓
3. 训练完成 → 点击"完成"按钮
   ↓
4. 指针推进，显示下次训练内容
```

### 切换计划流程
```
1. 进入计划管理页
   ↓
2. 点击要切换的计划
   ↓
3. 确认激活
   ↓
4. 重新初始化进度指针（可选：从头开始或保持进度）
```

## 七、注意事项

1. **进度指针需要初始化**：激活计划后，首次获取今日训练前必须先调用初始化接口

2. **完成训练即推进**：点击"完成"后，指针立即推进，下次打开显示下一项

3. **循环机制**：计划最后一天完成后，会自动回到第一天

4. **断练不重置**：只要不点击"完成"，无论过几天，打开都显示同一项

5. **单计划激活**：用户同时只能有一个激活计划，激活新计划会自动取消旧计划
